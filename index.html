Dns gör det möjligt för oss att hitta till webbsidor utan att behöva skriva in ip- adressen. Dens huvudroll är att besvara förfrågningar från datorn så att den kan hitta till den ip-adress webbläsare är kopplat till. .

(Det som händer mellan klient och server kan kortfattas med fyras steg:)
  • Steg ett: klient omvandlar url koden till server i adress genom dns.
  • Steg två: klient ansluter till server med hjälp av HTTP eller HTTPS och begär index.html-filen.
  • Steg fyra: webbläsare tolkar index.html-filen, tillämpar stilar från styles.css och visar bilden från example.jpg.

    Url (https://medieteknik.lnu.se:443/wt1/index.html#intro)består av fem delar:

  • Https som anger vilken protokoll som används för kommunikation.
  • Medieteknik är dns som översätter ip-adress från server till webbsidan. lnu.se som är under domän.
  • 433 är porten på webbsidorna som webbläsaren ska ansluta till. 433 är den standardporten till Https kommunikation.
  • Wt1/index.html är sökvägen som anger den specifika platsen och filen som serven begär. Den kan delas upp i två bitar wt1 som är mappen, vilken betyder den specifika platsen och index.html är filen.
  • #intro är den fragment som pekar till en viss sida av webbläsaren, vilken betyder att om man skulle öppna hemsidan skulle den ta dig till den specifika sidan först
